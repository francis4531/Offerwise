Technical implementation
Every authenticated user is always a Buyer — the Buyer portal (settings.html) is the base account. Inspector and Agent each have a user_id foreign key with unique=True, meaning one Inspector record and one Agent record per user account. Contractor currently links by email (no user_id), so multi-persona detection for contractors uses email matching against the logged-in user. The role switcher in the top nav calls all three profile endpoints in parallel on login, shows only the roles the user holds, and lets them jump between portals in one click.
